Gst.Memory

GstMemory is a lightweight refcounted object that wraps a region of memory. They are typically used to manage the data of a Gst.Buffer.

A GstMemory object has an allocated region of memory of maxsize. The maximum size does not change during the lifetime of the memory object. The memory also has an offset and size property that specifies the valid range of memory in the allocated region.

Memory is usually created by allocators with a Gst.Allocator.prototype.alloc method call. When NULL is used as the allocator, the default allocator will be used.

New allocators can be registered with register. Allocators are identified by name and can be retrieved with find. Gst.Allocator.prototype.set_default can be used to change the default allocator.

New memory can be created with Gst.new_wrapped that wraps the memory allocated elsewhere.

Refcounting of the memory block is performed with gst_memory_ref() and gst_memory_unref().

The size of the memory can be retrieved and changed with Gst.get_sizes and Gst.resize respectively.

Getting access to the data of the memory is performed with Gst.map. The call will return a pointer to offset bytes into the region of memory. After the memory access is completed, Gst.unmap should be called.

Memory can be copied with Gst.copy, which will return a writable copy. Gst.share will create a new memory block that shares the memory with an existing memory block at a custom offset and with a custom size.

Memory can be efficiently merged when Gst.is_span returns TRUE.

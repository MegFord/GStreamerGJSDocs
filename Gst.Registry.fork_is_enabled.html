    // Gjs wrapper for gst_registry_fork_is_enabled()
}

By default GStreamer will perform scanning and rebuilding of the registry file using a helper child process.

Applications might want to disable this behaviour with the Gst.fork_set_enabled function, in which case new plugins are scanned (and loaded) into the application process.

Returns

true if GStreamer will use the child helper process when rebuilding the registry.
